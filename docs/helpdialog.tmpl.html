WHOI Quantification Results

Quantification Methods

In the chart we can compare the results for each of the methods tested. Each method tries to predict the prevalence for each class in all the files used for testing. Files are in the X-axis while prevalence is in the Y-axis. We have compared the following methods:

  • CC: Classify&Count
  • AC: Adjusted Count
  • PCC: Probabilistic Classify&Count
  • PAC: Probabilistic Adjusted Count
  • HDy: Method based on the Hellinger Distance
  • EM: Method based on the EM algorithm

Features used

Each of these methods has been tested using three different features sets:

  • NF: Normal features downloaded from IFCB dashboard
  • DF_resnet18: Attributes computed using resnet-18 trained for the ImageNet problem. No fine-tunning has been made to adapt the network to the plankton problem.
  • DF_resnet34:Attributes computed using resnet-18 trained for the ImageNet problem. No fine-tunning has been made to adapt the network to the plankton problem.
  • DFFT_resnet18_full: Attributes computed using resnet-18 trained for the ImageNet problem. All training images were used for fine tunning (1/2 million).
  • DFFT_resnet34_full: Same than before but using a network with 34 layers.
  • DFFT_resnet50_full: Same than before but using a network with 50 layers. This network generates 2048 features instead of 512.
  • DFFT_resnet101_full: Same than before but using a network with 101 layers.
